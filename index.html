project 1 : Kafka Message-Queue


"Kafka" คือ Message-Queue ที่คอยเชื่อมต่อระหว่าง Producer(ผู้ผลิต) และ Consumer(ผู้บริโภค) โดย Producer จะเป็นตัวผลิต Message ไปส่ง Message Queue ซึ่งจะรอให้ Consumer หยิบแต่ละ Message ไปใช้งาน

นอกจากนั้น Kafka ยังเป็นตัว Distributed System(ระบบกระจาย) ได้อีกด้วยคือเราสามารถสร้าง Producer หลายๆตัวขึ้นมาได้เพื่อให้สามารถรองรับการทำงานกับระบบที่มี Transaction(การติดต่อกัน) มากๆ เพื่อความเข้าใจง่ายจะขอยกตัวอย่างกับร้านอาหาร ตอนเริ่มต้นเราเปิดร้านอาหารลูกค้าก็ยังไม่มากเพราะยังไม่ค่อยมีใครรู้จักร้านเรา เราจ้างเด็กเสิร์ฟคนเดียวก็พอ แต่เมื่อเวลาผ่านไปร้านอาหารเราดังขึ้นมาติดมิชลินสตาร์คนเข้ามาอย่างล้นหลามการมีเด็กเสิร์ฟคนเดียวก็คงน้อยไปเพราะลูกค้าจะรอนานมากหรือคงจะไม่เข้าร้านเราอีกเลย ดังนั้นเราจึงแก้ปัญหาโดยการจ้างเด็กเสิร์ฟเพื่อให้สามารถให้บริการลูกค้าได้อย่างรวดเร็ว ซึ่งเด็กเสิร์ฟที่เพิ่มขึ้นนั้นก็เปรียบได้กับ Kafka ที่สามารถสร้างหลายๆ Producer เพื่อให้การทำงานรวดเร็วขึ้น

โดยสามารถใช่งานได้โดยการติดตั้งผ่านคำสั่ง npm install kafkajs

$ npm install kafkajs

ในโปรเจคนี้เราใช่ "Kafka " ในการจัดการกับ Message-Queue จำนวนมหาศาลที่เป็นผลลัพท์จากการ Query จากบทความข้างต้น Kafka นั้นเป็นตัวเลือกที่เหมาะสมที่สุดในการจัดการ ข้อมูลจำนวนมากแบบนี้

ด้วยความสามารถของ kafka ก็สามารถทำให้เราสามารถจัดการข้อมูลแบบนี้ได้ และนอกจาก Kafka นั้นยังมีระบบที่สามารถจัดการกับ Message-Queue จำนวนมากๆแบบนี้อีกเช่น
